# QFC 代币经济学设计

## 概述

QFC (Quantum-Flux Coin) 是 QFC 区块链的原生代币，用于支付交易费用、验证者质押、网络治理和生态激励。

## 基本参数

| 参数 | 值 |
|------|-----|
| **代币名称** | Quantum-Flux Coin |
| **代币符号** | QFC |
| **精度** | 18 位小数 |
| **初始总量** | 1,000,000,000 QFC (10亿) |
| **最大供应量** | 2,000,000,000 QFC (20亿) |
| **通胀模型** | 递减通胀 → 长期通缩 |

## 通胀与通缩机制

### 区块奖励 (通胀来源)

```rust
/// 区块奖励计算公式
fn block_reward(year: u64) -> U256 {
    let base_reward = 10 * ONE_QFC;  // 10 QFC

    // 每年减半，最低 0.625 QFC
    let halvings = year.min(4);
    base_reward >> halvings
}
```

| 年份 | 区块奖励 | 年增发量 (约) | 通胀率 (约) |
|------|----------|--------------|-------------|
| 第1年 | 10 QFC | 94,608,000 | 9.46% |
| 第2年 | 5 QFC | 47,304,000 | 4.31% |
| 第3年 | 2.5 QFC | 23,652,000 | 2.07% |
| 第4年 | 1.25 QFC | 11,826,000 | 1.01% |
| 第5年+ | 0.625 QFC | 5,913,000 | 0.49% |

*假设每3秒一个区块，每年约 10,512,000 个区块*

### 销毁机制 (通缩来源)

1. **交易费销毁 (20%)**
   - 每笔交易费的 20% 被永久销毁
   - 类似 EIP-1559 的 base fee 销毁

2. **惩罚销毁**
   - 验证者被罚没的质押金直接销毁
   - 双重签名: 50% 罚没并销毁
   - 其他违规按比例销毁

3. **治理销毁**
   - 失败提案的押金销毁
   - 恶意提案的惩罚销毁

## 验证者经济学

### 质押要求

| 参数 | 值 |
|------|-----|
| 最低质押 | 10,000 QFC |
| 最大活跃验证者 | 1,000 |
| 解锁等待期 | 7 天 |
| 委托最低金额 | 100 QFC |

### 收益来源

```
验证者收益 = 区块奖励 + 交易费收入
           = (区块奖励 × 70%) + (交易费 × 50%)  [生产者]
           = (区块奖励 × 30%) + (交易费 × 30%)  [投票者]
```

### 预估年化收益 (APY)

假设条件:
- 总质押量: 2亿 QFC (20% 流通量)
- 日交易量: 100万笔
- 平均交易费: 0.001 QFC

| 验证者质押量 | 出块概率 | 年化收益 (约) |
|------------|----------|--------------|
| 10,000 QFC | 0.005% | 8-12% |
| 100,000 QFC | 0.05% | 10-15% |
| 1,000,000 QFC | 0.5% | 12-18% |

*实际收益取决于贡献分数、网络活跃度和质押竞争*

### 委托机制

- 委托人可将 QFC 委托给验证者
- 验证者设置佣金比例 (0-20%)
- 委托人收益 = 验证者收益份额 × (1 - 佣金率)
- 委托人同样受到验证者惩罚影响

## 费用结构

### Gas 定价

```rust
// 动态 Gas 定价
struct GasPrice {
    base_fee: U256,      // 基础费用 (根据区块使用率调整)
    priority_fee: U256,  // 优先费 (给验证者的小费)
}

// 基础费调整算法 (类似 EIP-1559)
fn adjust_base_fee(parent_gas_used: u64, parent_gas_limit: u64, parent_base_fee: U256) -> U256 {
    let target = parent_gas_limit / 2;

    if parent_gas_used > target {
        // 区块过满，提高费用
        let increase = parent_base_fee * (parent_gas_used - target) / target / 8;
        parent_base_fee + increase.max(1)
    } else {
        // 区块未满，降低费用
        let decrease = parent_base_fee * (target - parent_gas_used) / target / 8;
        parent_base_fee - decrease.min(parent_base_fee)
    }
}
```

### 费用参数

| 操作 | Gas 消耗 | 费用 (1 Gwei base fee) |
|------|---------|------------------------|
| 简单转账 | 21,000 | 0.000021 QFC |
| ERC-20 转账 | ~65,000 | 0.000065 QFC |
| 合约部署 | ~500,000 | 0.0005 QFC |
| DEX Swap | ~150,000 | 0.00015 QFC |
| NFT 铸造 | ~100,000 | 0.0001 QFC |

## 激励对齐设计

### 验证者激励

```
贡献分数 = 质押权重 × 30%
         + 计算贡献 × 20%
         + 在线时间 × 15%
         + 验证准确率 × 15%
         + 网络服务 × 10%
         + 存储贡献 × 5%
         + 历史信誉 × 5%
```

**设计理由:**
- 多维度评估防止单一资本控制
- 长期参与者获得更高信誉权重
- 鼓励提供多样化网络服务

### 开发者激励

1. **Gas 费回馈**
   - 合约创建者可获得合约产生 Gas 费的 5%
   - 激励开发优质 DApp

2. **Grant 计划**
   - 生态系统基金资助优质项目
   - 季度评审，最高 100万 QFC/项目

3. **Bug 赏金**
   - 安全漏洞奖励: 1,000 - 1,000,000 QFC
   - 根据严重程度分级

### 用户激励

1. **早期用户空投**
   - 测试网活跃用户
   - 社区贡献者

2. **流动性挖矿**
   - DEX 流动性提供者奖励
   - 借贷协议激励

3. **推荐奖励**
   - 邀请新用户获得交易费返还

## 风险与对策

### 中心化风险

**风险**: 大户控制网络

**对策**:
- 贡献分数多维度评估 (非纯 PoS)
- 验证者数量上限 1000
- 质押上限 (单验证者最多 1% 总质押)

### 通胀风险

**风险**: 代币贬值

**对策**:
- 递减通胀模型
- 20% 交易费销毁
- 惩罚销毁机制

### 流动性风险

**风险**: 质押锁仓导致流动性不足

**对策**:
- 流动质押衍生品 (stQFC)
- 合理解锁等待期 (7天)
- 流动性储备
