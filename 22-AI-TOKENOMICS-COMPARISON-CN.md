# AI 链代币经济学对比与 QFC 启示

> 最后更新：2026-03-08 | 版本 1.0
> GitHub Issue: #4
> 作者：Alex Wei，QFC Network 产品经理

---

## 1. 概要

本报告对比各 AI 区块链项目的代币经济学模型，为 QFC 的代币经济设计提供参考。

**核心发现：**

- **没有任何 AI 链实现了自我可持续的收入**——所有项目都依赖代币发行来补贴提供者
- **燃烧-铸造均衡（BME）** 正成为新兴标准（Render 首创，Akash 正在采用，io.net 变体）
- Bittensor 的纯发行模型风险最高（零费用收入，100% 依赖发行资金）
- **Filecoin 的抵押模型**创造了最强的提供者对齐，但入门门槛高
- 所有项目的代币价格均与使用量脱钩（Akash：收入 +38%，代币价格 -89%）
- **io.net 拥有最佳原始收入**（年化约 1840 万美元），在计算网络中领先

**对 QFC 的建议**：采用 **BME + 抵押 + 减半** 混合模型——法币定价推理并燃烧使用量，矿工质押抵押品，比特币式发行计划传递稀缺性信号。

## 2. 逐项目分析

### 2.1 Bittensor (TAO)

**供应量**：2100 万硬顶（效仿比特币）。2025 年 12 月首次减半（7,200 → 3,600 TAO/天）。

**发行分配**：

| 接收方 | 份额 | 角色 |
|-----------|-------|------|
| 矿工 | 41% | 执行 AI 推理/训练 |
| 验证者 | 41% | 评分矿工工作（Yuma Consensus） |
| 子网所有者 | 18% | 维护激励机制代码 |

**Yuma Consensus（权重分配）**：
- 验证者提交权重向量，对每个矿工的表现进行排名
- 权重形成二维矩阵（验证者 UID × 矿工 UID）
- 激励评分是质押加权平均值——委托的 TAO 越多，影响力越大
- "裁剪"机制：高于共识权重的权重被向下修正，惩罚串谋行为

**Dynamic TAO / Taoflow（2025 年 11 月）**：
- 每个子网拥有自己的 Alpha 代币，在去中心化池中与 TAO 交易
- 发行基于质押的净 TAO 流入（而非代币价格）
- 净质押流入 → 更高发行量；净流出 → 减少/零发行

**收入**：直接收入为 $0。纯发行驱动。价值取决于子网效用创造的 TAO 质押需求。

**评判**：最具创新性但风险最高。机制复杂（Yuma、dTAO）但没有费用收入支撑长期发展。

---

### 2.2 Render Network (RENDER)

**供应量**：6.442 亿最大值。已从以太坊（RNDR）迁移至 Solana（RENDER）。

**燃烧-铸造均衡（BME）**：

```
用户支付 $100 渲染任务
  → 95% 自动购买 RENDER → 永久燃烧
  → 5% → 基金会

节点运营者完成任务
  → 获得新铸造的 RENDER 作为奖励
```

- **核心创新**：任务以法币定价，使用户免受代币波动影响
- 当燃烧量 > 铸造量时通缩；需求低时通胀

**发行计划（RNP-006）**：
- 第 1 年：约 910 万 RENDER 总量（约 76 万/月）
- 节点运营者：每月发行量的 50%
- 按 RNP-001 递减计划执行

**RNP-018（第 2 年）**：为 AI 计算单独分配发行量（与渲染区分）。

**收入**：2025 年渲染 2200 万帧（占历史总量 35%）。但日活跃用户从约 1,500 降至 <100。该模型仅在使用量增长时才通缩。

**评判**：BME 是正确的框架。但 Render 的日活暴跌表明该模型需要真实需求才能运转。

---

### 2.3 Akash Network (AKT)

**供应量**：通胀型，无硬顶。初始通胀率 54%，每 3.75 年减半。当前：年化约 16.27%。

**反向拍卖定价**：
- 租户设定最高价格
- 提供者竞价；最低出价者胜出
- 价格通常**比 AWS/GCP 便宜 70-85%**

**抽成率**：每笔租赁付款的 20% → 分配给 AKT 质押者。

**BME 转型（2026 年 Q1）**：社区已批准。模拟结果：按 336 万美元月计算量，每月燃烧约 210 万 AKT（约 98.5 万美元）。

**实际收入数据**：

| 时期 | 租赁收入 | GPU 使用量 | 备注 |
|--------|-------------|-----------|-------|
| 2025 Q1 | >100 万美元（环比 +38%） | 553 GPU (+54%) | 创纪录季度 |
| 2025 Q2 | 82 万美元（-20%） | 1.9 万新租赁（-59%） | 季节性下降 |
| 2025 Q3 | 85.17 万美元（+4%） | 稳定 | 复苏 |
| 年化 | ~340 万美元 | ~50% GPU 利用率 | 日均每 GPU $20 |

**关键教训**：AKT 价格同比下跌 89%，尽管收入在增长。**代币价值与网络使用量脱钩。**

**评判**：最佳的真实世界收入数据。反向拍卖对价格发现有效。BME 转型体现了成熟化。

---

### 2.4 io.net (IO)

**供应量**：8 亿硬顶，20 年递减通胀。37.5% 分配给发行。

**激励动态引擎（IDE）——2025 年大改版**：
- 将代币发行与实时计算需求挂钩
- 目标将流通 IO 减少 50%
- GPU 供应者按实际租赁小时数（而非闲置算力）成比例获得收益

**定价**：
- 由供需自动决定（GPU 规格、连接性、安全性）
- 用户可以法币、USDC 或加密货币支付——全部转换为 IO
- IO 支付：零手续费；其他代币：2% 手续费
- 声称比传统云降价 70%

**质押（时间锁证明）**：
- GPU 运营者必须质押 IO 才能参与
- 已质押运营者获得每小时区块奖励
- 因在线时间不达标和欺诈而被削减

**收入**：年化约 1840 万美元（2024 年 11 月）。32.7 万已验证 GPU。

**评判**：计算网络中收入最强。IDE 模型将发行与需求挂钩，设计精巧。

---

### 2.5 Filecoin (FIL)

**供应量**：20 亿最大值。双重铸造：基线铸造（38.5%）+ 简单铸造（16.5%）。

**抵押与质押（独有）**：

| 抵押类型 | 用途 |
|----------------|---------|
| 初始质押 | ~20 天预估区块奖励 + 存储质押 |
| 区块奖励作为抵押 | 75% 在 180 天内线性释放 |
| 交易提供者抵押 | 存储交易的额外存款 |
| FIP-0077 存款 | 新存储提供者初始质押的 10%（每 10 TiB 约 4 FIL） |

**削减**：
- 扇区故障费：每次故障约 7 天区块奖励
- 故障检测费：额外惩罚
- 终止扇区费：惩罚提前退出

**关键洞察**：业内最激进的抵押模型。要求提供者锁定大量资金。提高了可靠性但造成高准入门槛——2025 年许多小运营者退出。

**评判**：业内最强的对齐机制。代价是准入门槛高。

---

### 2.6 Gensyn ($AI)

**供应量**：100 亿代币。2025 年 12 月代币销售（英式拍卖，$0.0001–$0.10）。3% 公开销售。

**效用**：计算支付、质押与验证、评估市场。

**状态**：尚无收入，非常早期。详细经济学尚未公布。

## 3. 代币设计模式对比

| 模式 | 描述 | 使用者 | QFC 相关性 |
|---------|-------------|---------|---------------|
| **硬顶 + 减半** | 比特币式稀缺性 | Bittensor (21M) | 强稀缺信号 |
| **燃烧-铸造均衡** | 法币定价，使用时燃烧，工作时铸造 | Render, Akash, io.net | 最利于用户采纳 |
| **通胀 + 抽成率** | 持续通胀配合费用捕获 | Akash（当前） | 简单但有稀释性 |
| **抵押/质押** | 提供者锁定代币作为利益绑定 | Filecoin, io.net | 最强对齐 |
| **反向拍卖** | 市场驱动定价，最低出价者胜出 | Akash | 良好的价格发现 |
| **动态发行** | 发行与实时需求挂钩 | Bittensor (Taoflow), io.net (IDE) | 减少浪费 |
| **削减** | 对停机/欺诈的惩罚 | Filecoin, io.net | 强制可靠性 |
| **工作代币** | 必须质押才能获得工作权利 | Filecoin, io.net | 抗女巫攻击 |
| **锁仓释放** | 延迟奖励分发 | Filecoin（75%/180 天） | 防止挖完即抛 |

## 4. 可持续性分析

| 项目 | 年收入 | 收入来源 | 依赖发行？ | 评级 |
|---------|---------------|----------------|---------------------|--------|
| **io.net** | ~1840 万美元 | GPU 计算租赁 | 过渡中 (IDE) | 中高 |
| **Akash** | ~340 万美元 | 计算租赁 | 过渡中 (BME) | 中 |
| **Render** | 未披露（日活下降） | 渲染/AI 任务 | BME 循环 | 中低 |
| **Filecoin** | 已建立但下降中 | 存储交易 | 抵押锁定 | 中 |
| **Bittensor** | $0 | 纯发行 | 100% 依赖 | 低 |
| **Gensyn** | 尚无收入 | N/A | 待定 | 未知 |

**普遍事实**：没有任何 AI 链实现了自我可持续收入。所有项目都依赖发行。问题在于哪种模型最能从发行补贴过渡到费用自持。

## 5. QFC 代币经济学建议

### 5.1 混合模型：BME + 抵押 + 减半

结合各项目的最强模式：

```
QFC 代币经济学：

┌─ 供应量 ────────────────────────────────┐
│  硬顶：待定（2100 万或更高）              │
│  发行：减半计划（BTC 式）                │
│  为长期持有者传递稀缺信号                 │
└──────────────────────────────────────────┘
        ↓
┌─ 推理费用流（BME）─────────────────────┐
│  用户为推理任务支付 $X                    │
│    → 85% 自动购买 QFC → 燃烧            │
│    → 10% → 网络金库                     │
│    → 5% → 协议开发                      │
│                                          │
│  矿工完成任务                            │
│    → 获得铸造的 QFC + 任务费用分成        │
│    → 75% 在 90 天内线性释放              │
│    → 25% 立即可用                        │
└──────────────────────────────────────────┘
        ↓
┌─ 矿工抵押（Filecoin 式）──────────────┐
│  矿工必须质押 QFC 才能加入网络           │
│    冷层：X QFC 最低                     │
│    温层：2X QFC 最低                    │
│    热层：5X QFC 最低                    │
│  削减：                                 │
│    错误推理：5% 质押 + 禁闭 6 小时       │
│    停机：每次违规 1% 质押                │
│    挑战失败：10% 质押                    │
└──────────────────────────────────────────┘
        ↓
┌─ 验证者激励 ──────────────────────────┐
│  PoC 加权发行份额                       │
│  Yuma 式裁剪反串谋                      │
│  委托支持（82% 分配给委托者）             │
└──────────────────────────────────────────┘
```

### 5.2 定价：法币计价 + 反向拍卖

借鉴 Render（法币定价）和 Akash（拍卖）：

```
1. 用户提交推理请求，附带最高价格（以 USD/稳定币计）
2. 任务路由器为该模型/层级找到合格矿工
3. 如果有多个矿工可用：
   → 反向拍卖：最低出价者胜出
   → 信誉加权：相同出价下 PoC 评分更高者优先
4. 付款自动转换为 QFC → 85% 燃烧
5. 矿工获得铸造的 QFC 奖励 + 剩余费用
```

**优势**：
- 用户以法币思维（利于采纳）
- 矿工在价格上竞争（高效市场）
- QFC 燃烧量与使用量成正比（通缩压力）

### 5.3 发行计划

| 年份 | 每日发行量 | 年度总量 | 备注 |
|------|---------------|-------------|-------|
| 1-4 | 待定基础费率 | — | 引导阶段，更高发行 |
| 5-8 | 前期的 50% | — | 首次减半 |
| 9-12 | 基础的 25% | — | 第二次减半 |
| ... | ... | ... | 每 4 年继续减半 |

**每区块分配**：

| 接收方 | 份额 | 理由 |
|-----------|-------|-----------|
| 矿工（推理） | 40% | 主要工作贡献者 |
| 验证者（共识） | 30% | 保护网络安全 |
| 金库 | 15% | 资助开发和拨款 |
| 质押奖励 | 15% | 激励代币锁定 |

### 5.4 动态发行调整（受 io.net IDE 启发）

```
If network_utilization > 80%:
    emission_multiplier = 0.8  // 需求高时减少发行（费用已足够）
If network_utilization < 20%:
    emission_multiplier = 1.5  // 增加发行以吸引矿工
Else:
    emission_multiplier = 1.0  // 标准费率
```

这可以在网络盈利时防止过度发行，在冷启动期提升激励。

### 5.5 矿工奖励锁仓释放（受 Filecoin 启发）

```
矿工完成推理获得 100 QFC：
  → 25 QFC 立即可用
  → 75 QFC 在 90 天内线性释放
  → 如果矿工在释放期间被削减：未释放奖励没收
```

**理由**：防止挖完即抛。使矿工与网络长期健康保持一致。

## 6. 费用对比与 QFC 定位

| 网络 | 推理成本（7B LLM，1K tokens） | 抽成率 | 支付方式 |
|---------|-------------------------------------|-----------|---------|
| Akash | ~$0.02-0.05（根据 GPU 定价估算） | 20% | AKT/USDC |
| io.net | ~$0.01-0.03 | 2%（非 IO） | IO/USDC/法币 |
| Bittensor | 免费（发行资助） | 0% | N/A |
| Render | 按任务法币定价 | 5%（基金会） | RENDER (BME) |
| **QFC（目标）** | **$0.005-0.02**（有竞争力） | **10%** | **QFC (BME) / USDC** |

QFC 应定价**比 Akash 低 30-50%** 以保持竞争力，依靠消费级 GPU 矿工实现（基础设施成本低于数据中心）。

## 7. QFC 的关键经验

### 行之有效的做法
1. **BME 法币定价**（Render）——用户无需理解代币
2. **反向拍卖**（Akash）——高效的计算价格发现
3. **抵押质押**（Filecoin）——提供者真正的利益绑定
4. **动态发行**（io.net IDE）——将奖励与真实需求挂钩
5. **减半计划**（Bittensor）——长期稀缺性叙事

### 行不通的做法
1. **纯发行无费用收入**（Bittensor）——没有外部需求则不可持续
2. **高准入门槛**（Filecoin 抵押）——排斥小矿工
3. **代币价格与使用量脱钩**（Akash）——DePIN 通病，目前无解
4. **无锁仓释放**——纵容挖完即抛行为

### QFC 特有考量
- **消费级 GPU 矿工**成本低于数据中心提供者 → 可提供更便宜的推理
- **AI 推理具有突发性**——需要动态定价和弹性矿工激励
- **模型多样性**创造天然的市场细分（见文档 #18）——不同模型层级不同定价
- **zkML 验证**（见文档 #20）增加成本开销——需纳入费用模型考虑
