# 意图驱动架构：研究与 QFC 影响分析

> 最后更新：2026-03-08 | 版本 1.0
> GitHub Issue：#9
> 作者：Alex Wei，QFC Network 产品经理

---

## 1. 执行摘要

意图驱动架构是从命令式交易（"先做 A 再做 B，支付 C"）到声明式意图（"我想要结果 X，愿意支付最多 C"）的范式转变。本报告评估意图驱动模式是否能改善 QFC 的 AI 推理请求/履行用户体验。

**核心发现：**

- **Anoma** 是最具雄心的意图中心协议——完整的操作系统，包含资源机器、求解器网络和异构共识（Typhon）。主网第一阶段于 2025 年 9 月上线
- **CoW Protocol**（月交易量 100 亿美元）证明求解器竞争在 DeFi 批量拍卖的生产环境中可行
- **UniswapX** 和 **1inch Fusion** 展示了基于荷兰拍卖的大规模意图履行
- **ERC-7683**（Across + Uniswap Labs）正在成为跨链意图标准
- **求解器中心化是一个现实问题**——在以太坊上，2 个构建者赢得超过 90% 的区块拍卖
- AI 推理请求天然适合映射为意图，但**求解器经济效益薄弱**（相比 DeFi，优化空间更简单）

**建议**：采用**意图模式**（声明式请求 + 竞争性履行），而非完整的意图基础设施。为复杂/批量推理任务实现轻量级"意图模式"，同时保留直接 TaskPool 用于延迟敏感型工作负载。

## 2. 主要意图驱动项目

### 2.1 Anoma 协议

**简介**：面向意图中心应用的分布式操作系统。用户表达期望结果，求解器寻找最优执行路径。

**核心组件**：

| 组件 | 角色 |
|-----------|------|
| **意图传播网络** | 广播意图；求解器发现匹配的对手方 |
| **求解器** | 匹配意图并产生平衡交易的第三方参与者 |
| **有效性谓词（VPs）** | 检查状态变更是否有效的无状态逻辑；可并行化 |
| **Anoma 资源机器（ARM）** | 带有资源模型的执行层（既非账户模型也非 UTXO 模型） |
| **Typhon** | 异构 Paxos 共识——多链原子结算 |

**资源模型**：资源是携带自身逻辑（有效性谓词）的一等原语。可以表示任何事物——代币、数字资产、能力。与编程语言理论中的线性类型相关（消耗而非复制）。

**状态**：
- 首个开发网：2025 年 1 月
- 主网第一阶段：2025 年 9 月（在以太坊上）
- 未来：隐私求解（FHE、MPC）、按需共识、"Chimera 链"

**Namada**（Anoma 的第一条链）：
- 主权 PoS L1，使用 CometBFT，2024 年 12 月主网上线
- 多资产隐私池（MASP）——将 Zcash Sapling 扩展到任意资产
- 隐私奖励：用户通过贡献隐私集赚取 NAM

### 2.2 Essential 协议

**简介**：以太坊上的声明式意图中心 L2。使用基于约束的编程替代命令式执行。

| 属性 | Anoma | Essential |
|----------|-------|-----------|
| 范围 | 完整操作系统 | 以太坊上的专注意图层 |
| 语言 | ARM 资源模型 | **Pint**（约束 DSL） |
| 隐私 | 内置（ZKPs、FHE） | 非主要关注点 |
| 共识 | 自有（Typhon） | 依赖以太坊 |
| 状态 | 主网第一阶段 | 预 Alpha，状态不明 |

**Pint 语言**：开发者定义允许哪些状态变更（约束），而非如何变更状态。求解器负责解决"如何"的问题。

**融资**：由 Archetype 领投的 1100 万美元 A 轮。

### 2.3 CoW Protocol（需求巧合）

**简介**：批量拍卖 DEX，用户意图被分组到约 30 秒的窗口中。求解器竞争以最优方式结算每个批次。

| 属性 | 值 |
|----------|-------|
| 月交易量 | 100 亿美元以上（2024 年末/2025 年初） |
| 机制 | 批量拍卖，统一清算价格 |
| 求解器网络 | DeFi 中最大的 |
| MEV 保护 | 相同代币对的订单以统一价格清算 |
| 创新 | 组合拍卖（每批次多个求解器） |

**工作原理**：同一批次、同一代币对的订单以统一价格清算——消除了批次内的三明治攻击和抢先交易。

### 2.4 UniswapX

**简介**：基于意图的交易，用户签署链下"交易意图"消息。荷兰拍卖价格发现。

| 属性 | 值 |
|----------|-------|
| 机制 | 荷兰拍卖——价格从目标价以上开始，随时间递减 |
| 填充者 | MEV 搜索者、做市商、链上代理竞争填充 |
| Gas 成本 | **用户零 Gas 费**，通过 Permit2 签名 |
| 逐笔处理 | 每个意图单独拍卖（不同于 CoW 的批量处理） |

### 2.5 1inch Fusion

**简介**：基于意图的兑换，使用经过验证的专业"解析器"。

- 每笔订单在解析器之间进行荷兰拍卖
- 三阶段执行，带托管机制
- **Fusion+**：跨链原子兑换
- 内置恢复：如果解析器无响应，时间锁到期后自动取消

### 2.6 Across 协议与 ERC-7683

**Across 协议**：基于意图的跨链桥接。
- 三层系统：报价请求 → 中继者竞争 → 结算验证
- 中继者使用自有资金填充订单；用户近乎即时获得资金
- V4（2025 年 7 月）：基于意图 + ZK 证明的通用架构

**ERC-7683**（跨链意图标准）：
- 由 Across + Uniswap Labs 联合编写
- 标准化：`CrossChainOrder`、`ResolvedCrossChainOrder`
- 结算接口：`IOriginSettler`、`IDestinationSettler`
- 实现跨协议的共享填充者网络
- 90 天内跨链交易量达 41 亿美元

## 3. 求解器/填充者网络

### 3.1 求解器工作原理

求解器接收用户意图，计算最优执行路径，竞争履行意图。它们支付 Gas 费、处理路由复杂性，并从用户可接受价格与实际执行价格之间的差价中提取利润。

### 3.2 求解器经济

- CoW Protocol：高级求解器团队年收入达数十万美元
- 收入 = 用户最高可接受价格与实际达成价格之间的差额
- MEV 提取被重新导向求解器竞争（理想情况下惠及用户）

### 3.3 中心化风险——关键警告

| 指标 | 值 |
|--------|-------|
| 区块拍卖赢家 | **2 个构建者赢得超过 90%** 的以太坊区块拍卖（2025 年初） |
| MEV 交易量 | 总计 5.6192 亿美元（2025 年），三明治攻击 = 2.8976 亿美元（51.56%） |
| 趋势 | 执行转向私有通道和工业化 MEV 供应链 |
| 效果 | 减少了用户端伤害，但**权力集中在更少的中间人手中** |

**反馈循环**：拥有更优延迟/订单流的公司更快积累资本，进一步主导市场。这是每个已部署的求解器网络中观察到的持续现象。

**缓解策略**：求解器轮换、保证金要求、最低求解器多样性阈值、批量拍卖（CoW 风格）。

## 4. 基于意图的 AI 推理分析

### 4.1 意图映射

AI 推理请求天然适合映射为意图模式：

| 意图概念 | QFC 推理等价物 |
|---------------|------------------------|
| 用户意图 | "用模型 X 分类这张图片，准确率 ≥ 阈值" |
| 约束条件 | max_cost、max_latency、min_accuracy、model_version |
| 求解器 | 矿工选择算法/撮合器 |
| 填充者/解析器 | 执行推理的矿工 |
| 荷兰拍卖 | 矿工间的价格发现（从最高价向下竞标） |
| 结算 | 推理交付证明 + 支付 |
| 有效性谓词 | 推理质量验证逻辑 |

### 4.2 当前模型 vs 意图模型

**当前 TaskPool（命令式）**：
```
用户创建任务 → 任务进入池 → 矿工领取 → 执行 →
提交结果 → 验证 → 支付
```

**意图模型（声明式）**：
```
用户表达意图（"推理 X，成本 ≤ Y，延迟 ≤ Z"）→
求解器竞争匹配最优矿工 → 获胜求解器路由 →
矿工执行 → 结果交付 → 结算
```

### 4.3 优势

1. **更好的用户体验**：用户指定结果，而非执行细节
2. **求解器竞争**：降低成本和延迟
3. **可组合性**：意图可以组合（"分类图片，如果类别=X 则运行模型 Y"）
4. **MEV 保护**：批量拍卖防止推理定价的抢先交易
5. **跨链**：借助类 ERC-7683 标准，推理意图可跨链履行

### 4.4 风险

1. **求解器中心化**：2-3 个精密求解器可能主导矿工选择，提取租金
2. **延迟开销**：意图传播 → 求解器匹配 → 矿工路由增加跳数，不如直接池领取
3. **复杂性**：完整的意图基础设施是重大工程开销
4. **薄弱的求解器经济**：用户最高价与矿工最低价之间的差价可能太小，不足以支撑求解器盈利（比 DeFi 路由的优化空间更简单）
5. **质量验证**：DeFi 意图有明确的成功标准（代币已交付）；推理质量更难以声明式验证

### 4.5 评估

| 因素 | 评分 | 备注 |
|--------|--------|-------|
| 架构适配度 | 中高 | 推理请求与意图映射良好 |
| 实施复杂度 | 高 | 完整的意图栈是重量级工程 |
| 用户体验提升 | 中等 | 主要优势：隐藏矿工选择的复杂性 |
| 求解器经济可行性 | 低-中 | 相比 DeFi 交易利润空间薄弱 |
| 中心化风险 | 高 | 在每个已部署的求解器网络中均有观察 |
| 延迟影响 | 负面 | 额外跳数影响实时推理 |
| 可组合性收益 | **高** | 多步骤 AI 流水线受益最大 |

## 5. QFC 推荐的混合方案

### 5.1 双路径架构

不要构建完整的 Anoma 风格意图操作系统。相反，将意图**模式**作为协议扩展来采用：

```
┌─────────────────────────────────────────────────────┐
│  用户 / DApp                                          │
│  选择：直接模式或意图模式                               │
├──────────────────────┬──────────────────────────────┤
│  直接模式             │  意图模式                      │
│  （延迟敏感型）        │  （成本优化型/复杂型）          │
│                      │                                │
│  Task → TaskPool     │  Intent → 撮合器               │
│  → 首个合格           │  → 约束条件评估                 │
│  矿工领取             │  → 矿工间荷兰拍卖              │
│  → 执行 → 验证       │  → 最优出价胜出                 │
│                      │  → 执行 → 验证                  │
│  延迟：<1 秒          │  延迟：5-30 秒                  │
│  成本：市场价          │  成本：优化后（更低）            │
├──────────────────────┴──────────────────────────────┤
│  共享层：AI 协调器、验证、结算                           │
└─────────────────────────────────────────────────────┘
```

### 5.2 直接模式（现有 TaskPool）

保留用于：
- 实时推理（聊天机器人、实时分类）
- 简单的单模型请求
- 延迟敏感型应用

无需修改——这是 QFC 的现有架构。

### 5.3 意图模式（新增）

新增用于：
- **成本优化的批量任务**："运行 1000 个嵌入，预算 = X QFC，截止时间 = 1 小时"
- **多模型流水线**："分类图片 → 如果是动物，运行物种检测 → 生成描述"
- **质量约束请求**："生成文本，困惑度 ≤ 阈值，成本 ≤ Y"
- **跨链推理**（见文档 #25）：来自外部链的意图 → QFC 履行

**意图结构**：

```move
resource InferenceIntent {
    id: UID,
    requester: address,
    // 内容
    model_id: Option<ModelId>,      // 指定模型或匹配任意能力
    capability: String,              // "text-generation"、"embedding" 等
    input_hash: Hash,
    // 约束条件
    max_cost: u64,                   // 最高费用（QFC）
    max_latency_ms: u64,            // 最大可接受延迟
    min_accuracy: Option<u64>,       // 最低质量阈值（0-100）
    // 组合
    next_intent: Option<UID>,        // 完成后链接到下一个意图
    condition: Option<String>,       // 条件路由逻辑
    // 生命周期
    deadline: u64,                   // 超过此时间戳后自动取消
    created_at: u64,
}
```

### 5.4 轻量级撮合器（非完整求解器网络）

**核心洞察**：对于 AI 推理，优化空间比 DeFi 路由更简单。轻量级"撮合器"就足够了——不需要带有传播和竞争的完整求解器网络。

```
撮合器逻辑：
  1. 接收 InferenceIntent
  2. 筛选合格矿工（模型支持、层级、可用性）
  3. 如果是单模型简单请求：
     → 匹配最便宜/最快的合格矿工（按 PoC 评分加权）
  4. 如果是多模型流水线：
     → 规划执行图
     → 将每个步骤分配给最优矿工
     → 协调顺序/并行执行
  5. 如果是成本优化批量任务：
     → 荷兰拍卖：向合格矿工广播，收集竞标
     → 接受满足约束的最低出价
     → 批量处理多个意图以提高效率
```

**为什么不需要完整的求解器网络**：
- AI 推理的求解器经济效益薄弱（简单匹配 vs 复杂的 DeFi 路由）
- 中心化风险高——2-3 个求解器会主导市场
- 传播/竞争的延迟开销影响实时用例
- 协议级撮合器避免了租金提取

### 5.5 非紧急推理的荷兰拍卖

借鉴 UniswapX/CoW Protocol 用于成本优化任务：

```
1. 用户提交意图："运行 100 个嵌入，最高 $0.50，截止时间 10 分钟"
2. 价格从用户最高价（$0.50）开始随时间递减
3. 矿工观察递减价格，在有利可图时出价
4. 第一个接受的矿工赢得该批次
5. 结果：用户支付低于最高价；矿工赚取高于最低价
```

**批次窗口**：将类似意图分组到 30 秒窗口中（CoW 风格），以提高批量效率。

## 6. 可组合的意图流水线

QFC 中意图的最高价值用例——多步骤 AI 流水线：

### 6.1 示例：基于情感的交易代理

```
意图流水线：
  步骤 1："获取提及 $TOKEN 的最新 100 条推文"
          → 外部数据预言机
  步骤 2："对每条推文运行情感分析"
          → QFC 嵌入模型 + 分类
  步骤 3："如果平均情感 > 0.7，生成买入信号"
          → QFC LLM 生成结构化输出
  步骤 4："如果有买入信号，在 DEX 执行交易"
          → 跨 VM 调用 EVM DeFi 协议

约束条件：总成本 ≤ 5 QFC，总延迟 ≤ 60 秒
```

### 6.2 示例：内容审核流水线

```
意图流水线：
  步骤 1："分类图片内容"
          → QFC 图片分类模型
  步骤 2："如果 NSFW 置信度 > 0.9，标记并停止"
          → 条件路由
  步骤 3："如果安全，生成替代文本描述"
          → QFC LLM 文本生成
  步骤 4："嵌入描述用于搜索索引"
          → QFC 嵌入模型

约束条件：准确率 ≥ 95%，成本 ≤ 2 QFC
```

这些流水线用命令式方式难以表达，但作为意图链则非常自然。

## 7. 对比：以交易为中心 vs 以意图为中心

| 属性 | 以交易为中心（当前） | 以意图为中心（拟议扩展） |
|----------|------------------------------|-------------------------------------|
| 用户指定 | 精确的执行步骤 | 期望结果 + 约束条件 |
| 执行路径 | 由用户固定 | 由撮合器/求解器优化 |
| 定价 | 用户设定精确费用 | 荷兰拍卖/竞争性竞标 |
| 可组合性 | 手动多交易协调 | 声明式流水线链 |
| 延迟 | 最小开销 | 额外的匹配步骤 |
| 复杂性 | 简单 | 较高（撮合器逻辑） |
| 最适用于 | 实时、简单请求 | 批量、复杂、成本敏感型任务 |

## 9. 核心要点

1. **采用模式，而非平台**——QFC 不需要 Anoma 的完整操作系统。意图/求解器模式作为协议扩展就足够了
2. **双路径设计**——直接模式（快速、简单）+ 意图模式（优化、复杂）。用户根据需求选择
3. **轻量级撮合器优于完整求解器网络**——避免中心化风险和租金提取；AI 推理匹配比 DeFi 路由更简单
4. **可组合流水线是杀手级功能**——声明式表达的多步骤 AI 工作流确实优于命令式协调
5. **非紧急任务的荷兰拍卖**——经过验证的模式（UniswapX、CoW），通过竞争性竞标实现成本节省
6. **求解器中心化是最大风险**——每个已部署的求解器网络都显示出权力集中。协议级撮合可以缓解这一问题
