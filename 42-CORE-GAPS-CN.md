# QFC 核心缺口：生或死的三个问题

> 最后更新：2026-04-13
>
> 状态：**专攻阶段**。如果这三个缺口在 8–12 周内都没有可验证的进展，这条链没有继续做下去的理由。

---

## 为什么写这份文档

过去几个月我们做了很多外围：NFT 市场、DEX、Bridge、Explorer、Faucet、Wallet、测试网 tx bot、Grafana 告警……这些让链"看起来活着"，但**没有一个**在触碰 QFC 的真实价值主张：

> QFC 不是另一条 EVM 链。它的存在理由是——**矿工跑 AI 推理换取区块奖励，用户/Agent 在链上付费调用推理**。

如果这个闭环跑不起来，我们只是又一条慢速 EVM 兼容链。这份文档锁定三个必须解的核心缺口。每一个都有：**成功判据 / 放弃判据 / 两周内的第一步**。时间盒 8–12 周——到期没进展就停掉 QFC，把精力换到更有牵引力的项目上。

---

## Gap A — Miner Onboarding：外部矿工跑 QFC 推理节点

### 问题
当前只有我们自己跑的 2 个 miner 在产生推理证明，3 个内部 node 在产块。没有任何外部 miner 加入。没有外部矿工，"去中心化推理网络"是空话。

缺失最刺眼的一块：**没有一条能让陌生人在 30 分钟内跑起 miner 的路径**。

- 没有 `qfc-miner` 的独立仓库或 release binary
- 没有 "System Requirements" 文档（最低 CPU/GPU/RAM/带宽）
- 没有一键启动脚本（`curl | sh` 或 `docker run`）
- 没有 miner 首日能看到奖励的 dashboard

### 成功判据（6 周内）
- [ ] **10 个外部 miner** 连续跑 ≥ 7 天产生有效推理证明
- [ ] 有人在 Telegram/Discord 自发问装机问题（说明 onboarding 到了"有摩擦但可入门"的阶段）
- [ ] 一条推文/一篇博客导流到 getting-started 页，转化率 > 5%（看完装上的占比）

### 放弃判据
- 6 周后外部 miner < 3 个，且新增 onboarding 询问 < 每周 1 人 → miner 这条路不成立，考虑把 QFC 重新定位成纯推理 API（不走去中心化矿工路线）

### 两周内第一步
1. 建立 `qfc-miner` 仓库，产出 x86_64 + arm64 的 Docker image（ghcr.io 托管）
2. 写 `MINER-QUICKSTART.md`：硬件最低要求、启动命令、如何看奖励、如何提问
3. 在 `getting-started.qfc.network` 挂一页，链接到 faucet、explorer、miner docs
4. 自己按 MINER-QUICKSTART 在一台干净 VPS 上走一遍，**不能有任何"灵感时刻"的步骤**

## Gap B — Inference Demand：谁在付钱调用 QFC 推理？

### 问题
即便有 100 个 miner，如果**没人调用**推理，miner 只是在消耗电产生对测试网毫无需求支撑的证明。长期看必须有付费需求，否则代币只能靠投机价值。

当前：
- 推理 API（`qfc_submitInference` 或类似）没有暴露为公开的、有 rate limit、能用 API key 付费的服务
- 没有 SDK snippet 让外部开发者一行代码调用
- 没有一个 real 用户 case 在生产上跑（AgentHub 还是设计文档，games 是示例项目）
- 对比 OpenAI/Replicate/together.ai 这些中心化 API，QFC 没有 latency 优势、没有价格优势、没有模型多样性——**差异化必须是"可验证"+"抗审查"，而这两个目前都没做给用户看**

### 成功判据（8 周内）
- [ ] 公开的推理 API endpoint，带计费（按 token 或按任务）、有 `requests_per_day` 指标上报
- [ ] 至少 **3 个非我们团队**的真实调用者（可以是小项目、hackathon 参与者，不是 bot）
- [ ] 做出一个 demo 能给投资人/用户讲：**"QFC 推理凭什么"** — 必须是 1 分钟内能懂的 pitch，且演示可验证性/抗审查的某一个具体场景

### 放弃判据
- 8 周后没有任何外部调用者 → 需求端没有 PMF，矿工侧的 supply 再多也是空转

### 两周内第一步
1. 把 `qfc_submitInference` 或等价接口包装成公开 REST API（`api.qfc.network/v1/inference`），走 Traefik+rate limit
2. 出 SDK 片段：JS/Python 各 20 行内完成"调用 + 拿到结果 + 验证证明"
3. 列 5 个**具体的可能 use case**（agent 决策、内容审核、游戏 AI、等等），挑 1 个自己做个 demo，公开评估是否打动目标用户
4. 找 3 个外部开发者聊"你会在什么情况下用一条可验证推理链？"—— 无实质答案也是信号

## Gap C — Economic Incentives：奖励模型是否自洽

### 问题
PoC 共识的多维度评分（质押 30%、计算 20%、在线时间 15%、验证准确率 15%、网络质量 10%、存储 5%、信誉 5%）在白皮书层面自洽，但**没验证过在实际分布下是否产生合理的 miner 经济收益**。

具体未回答的问题：
- 矿工每天真实能拿到多少 QFC？对应电费/硬件摊销是否回本？（需要估算，不是假设 token 价格）
- 如果外部 miner 涌入，奖励通胀如何控制？通胀率在测试网 → 主网过渡时怎么降档？
- 推理任务需求不足时，矿工"空转"的奖励来源是什么？（如果是纯通胀，则代币价值持续稀释）
- 惩罚机制（slashing）触发条件是否过严/过松？测试网上有没有真实触发过？
- 质押门槛对"让陌生人加入"的友好度是否合理？

### 成功判据（4 周内）
- [ ] 一份 **Miner Economics Model** 文档，基于当前测试网数据给出"单 miner 日收益 QFC / 日收益 USD 估算（在几个代币价格假设下）/ 保本点"
- [ ] 模型在白板上至少被 2 位外部懂区块链经济的人审过，没有明显逻辑漏洞
- [ ] 决定主网参数：初始通胀率、奖励分配曲线、质押最低门槛

### 放弃判据
- 4 周后经济模型还算不清楚，或者算出来"只有代币 > $10 才有人跑"→ 需求侧没到之前经济上不成立，先搁置

### 两周内第一步
1. 从当前测试网测数据导出：单 miner 日奖励均值、单 inference 任务耗时/gas、质押总量、当前活跃矿工数
2. 把 PoC 评分函数写成一个 Python notebook，输入外部 miner 加入数量 n = [10, 100, 1000]，输出每个矿工的期望收益
3. 找 1–2 个熟悉 PoS 经济学的人（过往做过 Cosmos / Ethereum 经济研究）付费 review，45 分钟就够

## 决策框架

三个缺口**不是并行三个**，是强依赖：

```
Gap C (经济自洽)
   ↓ 决定是否值得让外人加入
Gap A (Miner Onboarding) ←───── Gap B (Inference Demand)
                   互相托底：miner 需要需求来分钱，需求需要 miner 来证明可用性
```

- **Gap C 不通过 → 不要继续推 A 和 B**。经济不自洽意味着 miner 加入最后要么回不了本要么挤兑代币。
- **C 通过 & A 做完了 B 没做 → miner 空转，token 靠投机**。还不如不做。
- **C 通过 & B 做了 A 没做 → 需求打到一个空 supply 上**。不行。

所以顺序上 **C（4 周）→ A 和 B 并行（6–8 周）**。

---

## 里程碑与 Kill Switch

| 时间点 | 判断 |
|--------|------|
| **T+4 周** | Gap C 输出经济模型。如果模型显示不成立 → **停掉 QFC**，把精力换项目 |
| **T+8 周** | Gap A 有 ≥ 3 个外部 miner，Gap B 有 ≥ 1 个外部调用者 → 继续往前推 |
| **T+12 周** | Gap A 有 ≥ 10 miner，Gap B 有 ≥ 3 调用者，且有 1 个能打动陌生人的 demo → 宣告 PMF 初步验证，进入主网准备阶段 |
| **任何时候** | 如果 8 周内三个缺口都没有实质进展（不是 commit 数，而是外部人参与度）→ **停** |

---

## 不做什么

这段时间里，除非和上面三个 Gap 直接相关，**以下工作暂停**：

- 新的 DeFi 功能（lending 未部署就保持）
- 新的 NFT / 游戏前端功能
- Bridge 扩展到更多链
- AgentHub 的 UI 打磨（除非是作为 Inference Demand 的 showcase）
- Explorer / Wallet 的锦上添花

所有 PR 在进入之前先问：**这件事在帮 Gap A/B/C 吗？** 如果不是，延后。
